  <files>sorting/__init__.py, sorting/merge_sort.py</files>
  <action>
    Create sorting/__init__.py as an empty file (just `# sorting package`).

    Create sorting/merge_sort.py implementing:

    ```
    def merge_sort(arr: list) -> list:
    ```

    The implementation MUST follow iterative bottom-up with pre-allocated scratch buffer. Exact structure:

    1. Handle edge cases: if len(arr) <= 1, return list(arr) (copy, not original).

    2. Create a working copy: `work = list(arr)` — the original must not be mutated.

    3. Allocate scratch buffer: `scratch = [0] * len(work)` — allocated once, reused for all merges.

    4. Outer loop — doubling sub-array width:
       ```
       width = 1
       n = len(work)
       while width < n:
           lo = 0
           while lo < n:
               mid = min(lo + width, n)
               hi  = min(lo + 2 * width, n)
               if mid < hi:  # Only merge if right partition is non-empty
                   _merge(work, scratch, lo, mid, hi)
               lo += 2 * width
           width *= 2
       return work
       ```

    5. Inner merge function `_merge(arr, scratch, lo, mid, hi)`:
       - i = lo (left pointer), j = mid (right pointer), k = lo (write pointer into scratch)
       - While i < mid and j < hi: copy the smaller of arr[i], arr[j] into scratch[k]; advance i or j; advance k
       - Drain remaining elements from whichever side was not exhausted
       - Copy scratch[lo:hi] back into arr[lo:hi]

    CRITICAL: Do NOT use list slicing (arr[lo:mid]) anywhere in the merge function. Use index arithmetic exclusively. The scratch buffer must be reused across all merge calls — do not allocate a new list inside _merge.

    The function must return a new list. The input arr must be unchanged after calling merge_sort(arr).

    Add a module docstring: "Iterative bottom-up merge sort with pre-allocated scratch buffer. O(n log n) time, O(n) space."
  </action>
  <verify>

    3. No list slicing: grep the source for `arr[` and verify the only bracket accesses are index reads (arr[i], arr[j], arr[k]) not slice reads (arr[lo:mid])
    4. Non-mutation: x = [3,1,2]; merge_sort(x); assert x == [3,1,2]
