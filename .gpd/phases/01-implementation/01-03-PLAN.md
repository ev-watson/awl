    ```
    def radix_sort(arr: list) -> list:
    ```

    The implementation MUST follow LSD base-256 with backward scan for stability. Exact structure:

    1. Handle edge cases: if len(arr) <= 1, return list(arr).

    2. Validate inputs (for safety): all elements must be non-negative integers. A simple check:
       ```python
       # Optional but defensive: ensure non-negative integers
       # (Caller is responsible per IMPL-03, but guard against accidental misuse)
       ```
       Do NOT raise an error — just assume inputs are valid non-negative integers per the contract.

    3. Create working copy: `work = list(arr)`.

    4. Pre-allocate one output buffer: `output = [0] * len(work)`. This is reused across all 4 passes.

    5. For pass_index in range(4):  (pass 0 = LSB, pass 3 = MSB)
       ```
       shift = 8 * pass_index          # 0, 8, 16, 24
       RADIX = 256

       # Step A: count occurrences of each byte value
       count = [0] * RADIX
       for x in work:
           count[(x >> shift) & 0xFF] += 1

       # Step B: convert counts to prefix sums (cumulative counts)
       # After this, count[b] = number of elements with byte value <= b
       # (i.e., count[b] is the exclusive upper bound index for digit b)
       for i in range(1, RADIX):
           count[i] += count[i - 1]

       # Step C: place elements into output, iterating BACKWARD for stability
       for i in range(len(work) - 1, -1, -1):
           b = (work[i] >> shift) & 0xFF
           count[b] -= 1
           output[count[b]] = work[i]

       # Step D: copy output back to work for next pass
       work[:] = output
       ```

       After all 4 passes, work is fully sorted. Return work.

    CRITICAL: Step C MUST iterate backwards (range(len(work)-1, -1, -1)). Forward iteration breaks LSD stability. The backward scan ensures that elements with equal byte values in the current pass maintain their relative order from the previous pass.

    CRITICAL: count[b] in Step B is the EXCLUSIVE upper bound (the index AFTER the last element with byte b). Step C decrements count[b] before use, so the first element placed gets index count[b]-1 (the last slot for that byte value). This is correct Lomuto-style stable counting sort.

    CRITICAL: Do NOT allocate a new output list inside the pass loop. Reuse the single `output = [0] * n` allocated before the loop.

    Byte extraction verification:
    - Pass 0 (shift=0):  extracts bits 0-7  (values 0-255)
    - Pass 1 (shift=8):  extracts bits 8-15 (values 0-255)
    - Pass 2 (shift=16): extracts bits 16-23 (values 0-255)
    - Pass 3 (shift=24): extracts bits 24-31 (values 0-63 for range [0, 10^9-1], since 10^9-1 < 2^30)

    For 10**9 - 1 = 0x3B9AC9FF:
    - Pass 0: 0xFF = 255
    - Pass 1: 0xC9 = 201
    - Pass 2: 0x9A = 154
    - Pass 3: 0x3B = 59

    The count array for pass 3 will have zeros for entries 60-255. This is correct — those values don't exist in the range.

    Add module docstring: "LSD (Least Significant Digit) radix sort, base 256, 4 passes for integers in [0, 10^9-1]. Uses counting sort with backward scan for stability. O(4n) time, O(n) space."
