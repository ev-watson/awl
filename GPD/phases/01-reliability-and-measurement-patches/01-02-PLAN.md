<objective>
Replace blended token-cost accounting in tally.py and dispatch_cost_report.py with split input/output token pricing. Default to Claude Opus 4.7 rates ($5 input, $25 output per MTok).

Purpose: Blended pricing underestimates real frontier spend by ~22% for Opus 4.7 (confirmed by pilot data). Split pricing is required for defensible savings measurement in Phase 4.
Output: Patched Python scripts, fixture test files with known-good arithmetic verification.
</objective>

  1. In experiments/tally.py, replace `--cost-per-mtok` with two new flags:

  2. Either remove the old `--cost-per-mtok` flag entirely, or keep it as a deprecated alias that sets both input and output to the same value and prints a deprecation warning to stderr.   3. Update render_report to accept input_cost_per_mtok and output_cost_per_mtok. For each awl record, read prompt_tokens and completion_tokens (already present in awl_arm.jsonl). Compute:
     ```
     input_cost = prompt_tokens * input_cost_per_mtok / 1_000_000
     output_cost = completion_tokens * output_cost_per_mtok / 1_000_000
     total_cost = input_cost + output_cost
     ```

  4. Update the avoided-cost line in the aggregate section to use the same split-rate arithmetic against baseline tokens. If baseline.csv only has total frontier_tokens, use a note indicating that baseline split data is needed for precise cost comparison. For now, apply a configurable or documented input/output ratio assumption, or compute cost only when split baseline data is available.

  5. Keep total_tokens visible in the per-task table for backward compatibility with old artifacts.

  6. Create experiments/test_tally.py with fixture tests:
     - test_split_cost_pilot_data: Create a temporary awl_arm.jsonl with the 3 pilot records (prompt_tokens: 4082/1931/2113, completion_tokens: 182/142/151). Verify split cost at $5/$25 produces:

       - Task 03: $0.014340
       - Total: $0.052505
     - test_blended_vs_split_difference: Show that blended $5/MTok on total_tokens=8601 gives $0.043005, which is 22% less than split -- demonstrating why split matters.
     - test_no_cost_flags: Verify that omitting cost flags produces a report without cost lines (no crash, no wrong defaults).
     - test_backward_compat: If old flag is kept as deprecated, verify it warns and works; if removed, verify clear error message.

  7. Run: python3 -m pytest experiments/test_tally.py -v (or python3 -m unittest if pytest unavailable). All tests must pass.
