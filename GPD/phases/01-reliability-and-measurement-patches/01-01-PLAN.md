  <files>src/dispatch.rs, src/main.rs, src/mcp_server.rs</files>
  <action>
  1. In src/dispatch.rs:1111, change the literal `2` to `1` in effective_max_attempts so the default for `apply && has_verify` is 1 attempt.

  2. Add `pub model: Option<String>` field to the DispatchOptions struct (after repomap_budget). Update DispatchOptions::new() to set `model: None`.

  3. In the dispatch flow (around src/dispatch.rs:573), change model selection from:
     `let model = defaults::configured_model_for_level(options.level)...`
     to:
     ```
     let model = match &options.model {
         Some(m) => m.clone(),
         None => defaults::configured_model_for_level(options.level)
             .map_err(|e| std::io::Error::new(std::io::ErrorKind::InvalidInput, e))?,
     };
     ```

  4. In src/main.rs parse_dispatch_options, add `--model` flag parsing following the existing `--level` pattern. Add `let mut model: Option<String> = None;` and the match arm. Set `options.model = model;` before returning.

  5. In src/mcp_server.rs server_tool_definitions, add `"model"` property to the awl_dispatch inputSchema.properties:
     `"model": {"type": "string", "description": "Override the model for this dispatch (e.g. qwen2.5-coder:14b). When omitted, uses the configured model for the given level."}`
     Also add model extraction from MCP parameters and set it on the DispatchOptions in the awl_dispatch handler.

  6. Add or update unit tests in src/dispatch.rs::tests:
     - test_effective_max_attempts_default_one: effective_max_attempts(None, true, true) == 1
     - test_effective_max_attempts_override_two: effective_max_attempts(Some(2), true, true) == 2
     - test_effective_max_attempts_no_apply: effective_max_attempts(None, false, false) == 1
     - test_effective_max_attempts_clamp_max: effective_max_attempts(Some(6), true, true) == 5
     - test_effective_max_attempts_clamp_min: effective_max_attempts(Some(0), true, true) == 1
     - test_dispatch_options_model_default_none: DispatchOptions::new(2).model.is_none()

  7. Audit and document: list all experiments/tasks/*/task.json files that set explicit max_attempts values. Record them in the SUMMARY. Do NOT change them (Phase 2).

  8. Run: cargo fmt, cargo clippy --workspace --all-targets -- -D warnings, cargo test. Fix any issues before proceeding. All must pass clean.
